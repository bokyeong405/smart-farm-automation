# predictive_pick — 무정지 예측 포착 (Path B)

> 원본 [`../../src/classify_tomatoes.py`](../../src/classify_tomatoes.py)(정지 방식) 기반 개선. 원본은 보존, 여기서 새로.

## 문제
원본은 ripe 감지 시 컨베이어 벨트를 **정지** → 픽 → 재가동.
구동영상 측정상 **객체당 벨트 ~4초 정지**(픽 동작 ~3s + 재가동 대기 ~1s) → 처리량 손실.

## 개선 — 속도벡터 무정지 예측 포착(moving pick)
1. 연속 detection 으로 토마토 **픽셀 속도** `v=(Δu,Δv)/Δt` 추정
2. affine 선형부(2×2)로 **로봇 속도(mm/s)** 변환
3. **예측 위치 = affine(현재) + 로봇속도 × T_LEAD** (도달시간 2.0s + 통신 0.3s)
4. 벨트를 **멈추지 않고** 예측 좌표로 픽 (벨트 일정 속도라 선형 예측이 잘 맞음)
- 흡착식 그리퍼: 여닫기 없음 → 접촉 즉시 흡입.

## 파일
| 파일 | 내용 |
|---|---|
| `classify_tomatoes_predictive.py` | 예측 포착 노드 (원본 구조 + `[PREDICTIVE]` 표시 추가부) |
| `sim_test.py` | 하드웨어 없이 시간개선율·위치오차 검증 |

## sim 결과 (입력값: 🎥영상측정 / ✅코드 / ❓가정)

**A. 시간 개선율** (N=10, 정지 오버헤드 4s/객체)

| 객체 간격 | 정지 방식 | 무정지 예측 | 개선율 |
|---|---|---|---|
| 2s | 60s | 30s | **50%** |
| 3s | 70s | 30s | **57%** |
| 5s | 90s | 50s | **44%** |
| 8s | 120s | 80s | **33%** |

→ 객체가 분리되어 매번 정지·재가동이 발생하는 시나리오에서 **약 30~57% 시간 단축**.
※ 객체가 연속(시야에 계속 ripe)이면 정지방식도 계속 멈춰 둘 다 로봇 병목 → 개선 미미.

**B. 위치 정확도** (affine 스케일 0.538 mm/px, T_LEAD 2.3s)

| 벨트 속도 | 무예측 오차 | 예측(추정10% 오차) |
|---|---|---|
| 40mm/s | 92mm (통째 빗나감) | **9.2mm** |

→ 무예측은 `벨트속도×2.3s` 만큼 빗나가고, 예측 시 **속도추정 오차분만** 남음.
※ 벨트 mm/s 는 가정값(코드엔 스텝레이트 1250/s만). 영상 광류로 실측 시 교체.

## 정직 범위 (면접 방어선)
- ✅ "무정지 예측 포착 **로직 구현 + 시뮬 검증**"
- ✅ "정지 대비 시간개선율·위치오차를 **시뮬레이션으로 정량화**"
- ❌ "**실로봇**에서 무정지 수확 완료" (ROS2·로봇 없음 → 미실행)
- 향후: 하강 중 재예측(시각 서보잉)으로 정확도 추가 개선.
